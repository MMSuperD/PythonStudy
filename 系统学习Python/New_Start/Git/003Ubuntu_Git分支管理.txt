一.分支管理
    1.分支概念
      相当于重新开了一条指针路线就像在十字路口分开一样.
    2.分支的基本命令
      2.1 查看分支
          git branch
      2.1 创建分支
          git branch 分支名
      2.2 切换分支
          git checkout 分支名
      2.3 创建并切换分支
          git checkout -b 分支名
      2.4 合并分支 (合并分支到当前分支)
          git merge 分支名
      2.5 删除分支
          git branch -d 分支名

    3.分支冲突
      两个分支修改了同一个文件,会导致分支冲突

    4.分支管理策略
      4.1 合并的时候如果可以,执行快速合并
      4.2 禁止快速合并
          git merge --no-ff 分支名

    5.bug分支
      5.1 保存当前工作状态
          git stash
      5.2 切换到bug分支,创建并切换到临时分支修复 bug

      5.3 在临时分支上修复bug 后 把临时分支合并到bug分支上面(采用 --no-off 禁止快速合并模式)并提交
          git merge --no-ff -m '修复bug001分支上面问题' bug001

      5.4 删除临时分支
          git branch -d 分支名

      5.5 切换回工作分支
          git checkout 分支名

      5.6 在工作分支上面恢复以前的工作状态
          git stash pop
